Plan: Galaxy- & System-Neuaufbau (Neustart)

Stand: 2025-12-30

Kurzbeschreibung
- Wir verwerfen den bisherigen Galaxy-/Systemaufbau komplett.
- Neue Basis: Galaxy mit Spiralarme(n), schwarzes Loch im Zentrum, Zellgitter als Grundstruktur.
- Systeme: Eine Sonne mit Planeten und Orbits (deterministisch aus Seeds).
- Sky-Sphere: Sterne werden aus der Galaxy-Position im 3D-Raum um die aktuelle Zelle projiziert.

Zielbild
- Eine einheitliche Welt: Galaxykarte, Zellen und Spielwelt nutzen dieselben Daten und Koordinaten.
- Systemwechsel folgt eindeutig der Zelle (kein separater "Systemmodus").
- Sky-Sphere zeigt stets eine konsistente Projektion der Galaxy um die aktuelle Zelle.

Kernkonzepte
1) Galaxy als 3D-Zellgitter
   - Zellgroesse = cell_size_map (Map-Einheit).
   - Zell-ID = floor(map_pos / cell_size_map).
   - Zellzentrum = (cell + 0.5) * cell_size_map.
   - Zell-Seed = Hash(cell_xyz, galaxy_seed).
   - Jede Zelle ist deterministisch: System oder leer.

2) Galaxy-Generator (Spiralarme + Schwarzes Loch)
   - Basis: Spiralarm-Verteilung im XZ, vertikale Streuung in Y.
   - Black-Hole-Zone: Innenradius ohne Systeme + weiche Uebergangszone.
   - Dichte-Funktion: abhaengig von Radius, Arm-Phasenwinkel, Streuung.
   - Zellentscheidung: Aus Zell-Seed wird entschieden, ob die Zelle ein System enthaelt.
   - Systemposition in Zelle: Jitter um Zellzentrum (max. definierter Anteil der Zellgroesse).

3) System-Generator (Sonne + Planeten)
   - system_seed = Zell-Seed (oder abgeleitet daraus).
   - Sternklasse: Farbe, Groesse, Emission.
   - Planeten: Anzahl, Orbit-Radien, Inclination, Umlaufgeschwindigkeit.
   - Alle Parameter deterministisch aus dem Seed (gleicher Seed = gleiches System).
   - system_radius = max(Orbit + Planetenradius), damit die System-Ansicht/Skalierung
     den tatsaechlichen System-Extent nutzt (kein ungenutztes Padding).

4) Systemwechsel (Zellbasiert)
   - Aktuelle Zelle aus player_abs -> map_pos -> cell_id.
   - Wenn Zell-ID wechselt:
     - System in neuer Zelle laden (falls vorhanden).
     - Leere Zelle: definierter Basissatz (z.B. Asteroidenfeld + Ambient-Light).
   - Hysterese an Zellgrenzen, um Flackern zu vermeiden (Wert definieren).

5) Galaxykarte (UI/Map)
   - Map zeigt exakt dieselben Zellen/Systemdaten wie das Gameplay.
   - LOD/Zoom steuert nur die Darstellung, nicht die Datenbasis.
   - Auswahl eines Systems = Fokus auf Zellzentrum (kein "Moduswechsel").

6) Sky-Sphere (Projektion der Galaxy um die aktuelle Zelle)
   - Quelle: Systeme im Radius um die aktuelle Zelle (z.B. N Nachbarzellen).
   - Richtung = (system_pos_map - cell_center_map) normalisiert.
   - Projektion auf Einheitssphaere, dann Skalierung auf Sky-Sphere.
   - Helligkeit/Groesse nach Distanz (weicher Falloff).
   - Aktualisierung: bei Zellwechsel (nicht pro Frame), optional glattes Crossfade.

7) Daten & Determinismus
   - Ein Generator fuer Galaxy + System, wiederverwendbar in Map und Gameplay.
   - Keine doppelten "Scale"-Definitionen; eine zentrale Quelle.
   - Savegame speichert mindestens: galaxy_seed, aktuelle Zelle, player_abs, floating_origin_offset.

Festlegungen (aus Rueckmeldung)
1) Zellgroesse (World): 1.000.000 Einheiten pro Zelle als Zielgroesse.
   - world_to_map_scale = 0.1 / 1_000_000 (0.0000001).
   - Daraus folgt: cell_size_map = cell_size_world * world_to_map_scale = 0.1 Map-Einheiten.
   - Hinweis: Floating Origin beibehalten, damit grosse Distanzen keine Praezisionsprobleme verursachen.
2) Spiralarm-Parameter: Startwerte aus der aktuellen Galaxy ableiten.
3) Black-Hole-Radius + Falloff: aus der aktuellen Galaxykarte ableiten.
4) System-Dichte pro Zellring: aus der aktuellen Galaxy ableiten.
5) Sky-Sphere: so gross/weit, dass Systemobjekte sichtbar bleiben und die Sky-Sphere nichts ueberdeckt.
   - Rendering-Hinweis: Sky-Sphere immer im Hintergrund rendern (kein Depth-Test/Write).
   - Vorschlag: Sky-Sphere-Radius = 0.95 * Kamera-Far, zentriert auf Spieler.
   - Stern-Sampling: Radius 3 Zellen, Falloff mit smoothstep bis Rand.
6) Hysterese: 3% der Zellgroesse an Zellgrenzen.

Ableitung aus aktueller Galaxy (konkrete Quelle)
- Quelle: scripts/galaxy_service.gd (aktuelle GalaxyService-Parameter).
- Spiralarm-Form/Verteilung uebernehmen:
  - arm_count, spiral_tightness, arm_spread, interarm_fraction,
    arm_width_multiplier, interarm_width_multiplier, radial_power,
    core_fraction, core_radius, vertical_spread, color_variation.
- Black-Hole-Zone uebernehmen:
  - black_hole_system_free_radius (Innenradius, normiert 0..1)
  - black_hole_density_falloff_width (Uebergangsbreite, normiert 0..1)
- Galaxy-Ausdehnung/Skalierung uebernehmen:
  - galaxy_position_scale, galaxy_position_bias, galaxy_vertical_bias.
- System-Dichte pro Zellring ableiten:
  - Nutze system_count + Verteilung der aktuellen Systeme (normalized_distance)
    als Histogramm ueber Radialringe.
  - Berechne pro Ring: p(cell_has_system) = systems_in_ring / cells_in_ring.
  - cells_in_ring = ring_area / (cell_size_map^2) * height_factor,
    height_factor aus vertical_spread.
  - Ergebnis: Kurve p(r) fuer die Zellentscheidung im neuen Generator.

Definitionen (Arbeitsbegriffe)
- Map-Pos: Position im Galaxy-Koordinatensystem.
- Cell-Pos: Diskrete Zell-ID (Vector3i).
- Cell-Center: Map-Pos des Zellzentrums.
- World-Pos: Absolute Spielwelt-Koordinaten (mit Floating Origin).
